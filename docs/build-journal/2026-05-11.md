## Session 5 — Recommendation Agent + orchestrator end-to-end

Built `agents/recommendation.py`, the deterministic `orchestrator.py`, and the `pmi-report` CLI. 11 tests pass, ruff clean.

End-to-end pipeline on the drone RFT:
- 71.2 s total · 41,981 in / 6,757 out tokens · ~$1.14 on Opus 4.7
- 3 audit records — one per agent — fully reconstructable

The Recommendation Agent produced 4 scope flags (bundling_risk + unusual_sample_requirement at high; qualification_overreach + spec_drift_from_market at medium) and 3 closed-taxonomy recommendations (SPLIT_TENDER, TIGHTEN_QUALIFICATION, PIN_RFI_FIRST). Executive summary is 118 words, leads with the dominant risk, cites the forecast range explicitly.

Two findings worth surfacing in the Declan demo:
- The agent stretched `TIGHTEN_QUALIFICATION` to mean "rework the qualification rules" (it actually proposed loosening the four-formats rule). The closed taxonomy needs a `LOOSEN_QUALIFICATION` or `REVIEW_QUALIFICATION` kind in the next iteration — a self-discovered finding.
- The Historical Analyst returned 12-30 / high on the standalone run earlier today, but 15-30 / medium in the pipeline run. Same prompt, same data, different result. The validation grid (session 6) is the discipline that catches drift like this.
