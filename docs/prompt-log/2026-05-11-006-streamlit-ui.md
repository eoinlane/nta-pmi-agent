## Design choices worth flagging

**Two operating modes, cached default.** The sidebar defaults to "Cached report" (instant, demo-safe) rather than "Live pipeline run" (~70 s, ~$1.14, depends on Anthropic API and network). For Declan's demo the cached path is the safe primary; live mode is the dramatic option if everything is going well.

**Audit tab pulls from `audit.db` for the specific record IDs in the loaded report.** Token counts and durations come back as expandable rows. This is the responsible-AI exhibit made visible — every claim in the report ties back to a logged call.

**Severity / priority badges** use traffic-light dots (🔴 🟡 🟢) inside Streamlit `st.container(border=True)` cards. Visual scan in seconds: high-severity items pop.

**Validation grid is a sidebar action, not auto-applied.** A report has one or more candidate actuals files; the user picks one and clicks "Score against actuals". Keeps the report-loading and validation flows distinct — important if the same report is being shown to someone for the first time and you want to *not* spoil the predicted-vs-actual reveal.

**PDF export uses pandoc + prince.** CLAUDE.md notes that prince is the project's existing rendering workflow; both are present on this machine. The engine fallback chain ensures the code keeps working if someone clones the repo without prince installed.
